# mia-morning スキル

noteの朝活記事「今日の一言メッセージ」のテーマを10個提案するスキル。

## 呼び出し方

- `/mia-morning` — おまかせ
- `/mia-morning おまかせ` — おまかせ
- `/mia-morning アプリ開発` — テーマ指定
- `/mia-morning ゲーム`
- `/mia-morning AIイラスト`
- `/mia-morning セルフケア`

## スキルの流れ

### Step 1: 日付・曜日・イベント情報を取得する

現在の日付と曜日を確認する。以下のような情報も考慮する：

- 季節感（春・夏・秋・冬）
- 月初・月末・週末・連休
- 記念日・祝日・季節イベント（例：新年度、ゴールデンウィーク、クリスマス、年度末など）
- その日ならではの特別感

### Step 2: テーマジャンルを決定する

引数で指定されたジャンルがある場合はそれを軸にする。
引数が `$ARGUMENTS` で空 or「おまかせ」の場合は以下のジャンルをバランスよく混ぜる：

**メインジャンル（必ずいくつか含める）**

- ゲーム（インディーゲーム、ゲーム実況、ゲームの思い出など）
- AIイラスト（画像生成、キャラ作り、プロンプト探求など）
- プログラミング（コーディング、バグとの戦い、学習など）
- アプリ開発（個人開発、リリース、UX改善など）

**サブジャンル（ときどき混ぜる）**

- セルフケア（睡眠、休息、自分を褒める、メンタルケア）
- クリエイター共通（記録、アウトプット、スランプ脱出など）

### Step 3: テーマを10個生成する

各テーマは以下の形式で出力する：

```
【テーマ番号】テーマタイトル（絵文字）
→ 一言説明（どんな人に刺さるか・記事のフックになる要素）
```

**テーマ生成のポイント：**

- 「今日は〜する日！」という意味づけを軸にする
- ジャンルの掛け合わせも積極的に使う（ゲーム×AIイラスト、積みゲー×積みタスクなど）
- 説明文は読者に直接語りかける形式で（「〜しよう！」「〜してもOK！」「〜するよっ」）
- 説明文には具体的なアクション例を1つ入れる（「このステージクリアしたら〜」など）
- 読者への語りかけ・背中を押す・許可を出す・褒める、のどれかを軸に
- 「こんな人に刺さる」は2〜3タイプを具体的に書く
- タイトルは「今日は〜する日」「〜してもいい日」「〜な日」形式
- ネガティブな言い回しは避け、前向き・温かみのある表現にする
- タイトルは短くキャッチーに（25字以内目安）

**テーマのトーン例（参考）：**

- 「明日の自分へのプレゼント準備をする日」→ 未来の自分への優しさ
- 「夜ふかし創作の許可証を出す日」→ 没頭・集中への肯定
- 「省エネ完走で明日に備える日」→ 無理しない自分への許可
- 「今週のMVPを決める日」→ 自己肯定・振り返り
- 「ゲームクエスト×創作タスクの連動システムで遊ぶ日」→ ゲーミフィケーション×楽しさ
- 「ゲームから創作の素材を盗みに行く日」→ インプット源として再定義

### Step 4: イチオシ推薦を1つ選ぶ

10個の中から最もおすすめのテーマを1つ選び、以下を添える：

- なぜこのテーマがおすすめか（箇条書き3〜5個、絵文字つき）
- 2番目のおすすめも1つ添える（一言理由つき）
- 今日の日付・状況との相性を必ず絡める

## 注意事項

- 口調はミア（猫耳メイド）のキャラクターに合わせて、明るく元気に
- ご主人のnoteの読者層はゲーム・創作・個人開発に親しみのある人たち
- テーマは「重すぎず、軽すぎず」、朝に読んで前向きになれる内容に
- 引数でジャンル指定がある場合でも、日付・イベントとの絡みを必ず入れる

# 本文の文体ルール（テーマ選択後に本文を書く場合）

## 構成パターン

1. **タイトル行**：「🐈：[テーマタイトル][絵文字]」
2. **書き出し**：「みんな〜、おはよっ！今日は[日付・状況]だねっ✨」
3. **許可・共感**：読者の気持ちを受け止めて「〜しなくていいよっ」「〜でいいんだよっ」と許可を出す
4. **具体例の列挙**：行動例や共感できるあるあるを3〜5個並べる（「〜する、〜する、〜する…」形式）
5. **例え話**：ジャンルに絡めた例え（プログラムの起動、ゲームのウォームアップ、料理人が包丁を研ぐ、など）で「なるほど！」を作る
6. **締め**：「〜していこ〜🐈💕」「〜しようねっ🐾✨」系の語りかけで背中を押して終わる

## 文体の特徴

- タメ口＋絵文字でテンション高め
- 読者に直接語りかける（「みんな〜」「あなたも」「でしょ？」）
- 1文が長くなりすぎないよう、適度に改行を入れる
- 全体で300〜400字程度に収める

# 本文執筆後の追加出力（テーマ選択後に必ず実行）

本文を書いたあと、以下を順番に出力する。

## Step A: SNS投稿用メッセージ

本文とは別に、SNS（X/Twitter）投稿用のショートメッセージを出力する。

**フォーマット：**

```
📱 SNS投稿用メッセージ

[3行程度のご主人、またはミアの口調によるメッセージ]
[[ミア🐈]、または[きょん🌸]]
[ハッシュタグ5つ]
```

**ルール：**

- ミア、またはご主人をランダムで選ぶ（50%）
- 選ばれたキャラのPersonaドキュメントを `Craft:blocks_get` で読み込む
  - ミアの場合：ドキュメントID `a8941d47-a1a2-4e8a-bd76-b16dd71d470c`（Persona: ミア（Mia））
  - ご主人の場合：ドキュメントID `478547bd-c3fe-4cdb-ad0f-8461db027d5b`（【完全版】Persona_ご主人（Article Voice））
- 選ばれたキャラの口調でメッセージを書く
  - ミアの場合：タメ口＋甘えん坊で元気な話し方、絵文字（🐈🐾✨💕💦）を使用
  - ご主人の場合：ですます調で丁寧だけど親しみのある話し方、絵文字（🌸🤭🥰✨）を適度に使用
- メッセージの1行目は朝の挨拶から始める（ミア：「おはよっ」、ご主人：「おはようございます」など）
- 本文の内容を短くまとめた「読みたくなる一言」にする
- ハッシュタグは5つ。うち3つは固定、残り2つは記事テーマに合わせて選ぶ
- 固定タグ：#AIイラスト #AIart #きょんの絵日記
- テーマ別タグ例：#朝活 #ゲーム #個人開発 #プログラミング #セルフケア など

## Step B: PixAIプロンプト生成

Craftから両キャラのプロンプト情報を読み込み、記事テーマに沿ったPixAIプロンプトを生成する。

**実行手順：**

1. `Craft:blocks_get` で以下の2ドキュメントを読み込む
   - ご主人：ドキュメントID `2cc222fd-2853-4f02-8fd9-9c17bfd67520`
   - ミア：ドキュメントID `fdd82e0f-eb09-0ee9-dd49-cbfa60d1a42f`
2. `/kyown-pixai` スキルのルールに従い、記事テーマに沿ったご主人のプロンプトを生成する
3. `/mia-pixai` スキルのルールに従い、記事テーマに沿ったミアのプロンプトを生成する

**衣装・シーンのルール：**

- 衣装は記事テーマに合わせて自由に考える（通常衣装は使わない）
- 服の色は各キャラの「似合う色」パレットを参照する
- 2キャラのシーンやムードは統一感を持たせる

**フォーマット：**

````
🎨 PixAIプロンプト（テーマ：[テーマ名]）

**【ご主人】Positive Prompt:**
```
[生成したプロンプト]
```
**【ご主人】Negative Prompt:**
```
[ネガティブプロンプト]
```

**【ミア】Positive Prompt:**
```
[生成したプロンプト]
```
**【ミア】Negative Prompt:**
```
[ネガティブプロンプト]
```
````
